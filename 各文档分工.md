#  Assets 文件夹详解（你列的那一组 + System/Gameplay/Utils/详细含义）

下面我把每个文件夹该放什么、命名建议和注意事项写清楚，直接发给美术和程序都能看懂。

```
Assets/
├─ Art/
├─ Audio/
├─ Prefabs/
├─ Scenes/
├─ Scripts/
│   ├─ System/
│   ├─ Gameplay/
│   ├─ UI/
│   └─ Utils/
└─ UI/
```

## `Assets/Art/`

**放美术资源（图像）**

* `Art/Backgrounds/` 背景图（1920×1080）
* `Art/Characters/` 角色立绘、精灵帧（PNG）
* `Art/Props/` 场景物件（道具）
* `Art/UI/` UI 专用图（按钮、图标）
* `Art/Opening/` 开场专用图片序列（你那堆 jpg）

**格式与建议**：UI 图用 PNG（透明）；大背景可 JPG（更小）；单张尽量 < 2MB；命名统一如 `bg_start_01.jpg`、`char_main_idle.png`

---

## `Assets/Audio/`

**放音效与音乐**

* `Audio/BGM/` 背景音乐（长曲）
* `Audio/SFX/` 按键、点击、环境声等短音效
* `Audio/VO/` 语音（如果有）

**建议**：音频放 WAV（开发阶段好听）或 MP3（体积小）；文件命名 `bgm_mainloop.mp3`、`sfx_click.wav`

---

## `Assets/Prefabs/`

**预制体（可复用的游戏对象）**

* 例如按钮预制（含 Image + Button + 绑定脚本）、敌人预制、交互道具预制
* Prefab 可以在场景里多次复用，更新 prefab 会影响所有实例

---

## `Assets/Scenes/`

**场景文件（Unity 场景 .unity）**

* `Scenes/Opening.unity` 开场场景
* `Scenes/Game.unity` 主游戏场景
* `Scenes/Rules.unity` 规则/黑屏界面
  **注意**：多人同时修改同一场景容易冲突 → 尽量场景拆分（UI 独立场景/小场景分割）

---

## `Assets/Scripts/` （主脚本目录）

将脚本按职责分类放入子目录：

### `Scripts/System/`

* 全局管理器、单例类（GameManager、AudioManager、EscapeQuit）
* 常驻 Scene 的脚本

### `Scripts/Gameplay/`

* 游戏核心逻辑（解谜逻辑、关卡触发器、玩家控制）
* Platformer 有 `PlayerController`、`EnemyAI` 等

### `Scripts/UI/`

* UI 控制脚本（OpeningSequence、MenuController、DialogController）
* Button 事件绑定方法

### `Scripts/Utils/`

* 工具函数、扩展方法（如 `Singleton<T>`、存档工具、常用助手类）

**命名建议**：脚本类名与文件名一致，如 `OpeningSequence.cs` → class `OpeningSequence`

---

## `Assets/UI/`

**存放纯 UI 预制、UI 布局、TextMeshPro 资源**（可以与 Art/UI 合并，但建议分开）

* `UI/Prefabs/`：UI 预制（dialog box、item panel）
* `UI/Fonts/`：字体资源
* `UI/Layouts/`：UI 场景或 Canvas 布局保存

---

# 补充：关于场景冲突的额外建议

* **尽量不要多人同时编辑同一场景 `.unity` 文件**。如果必须→把场景拆成多个 additive scene（例如：Level_Base + Level_Entities + Level_UI），这样多人可以并行工作。
* **Prefab 化**：所有可交互物件做 Prefab，程序只改 Prefab，减少场景冲突。

---

# 附加小贴士（给你们的快速决策）

* **美术上传**：让美术只上传到 `Assets/Art/`，并用小分支 `asset/xxx` 来 push。提交说明写 `asset: add opening images`。
* **图片压缩**：大图交付时请导出 JPG（高质量但压缩），并在导入 Unity 前用工具（如 TinyPNG）压缩到 <2MB。
* **开场点击逻辑**：把开场图片做成数组（Sprite[]），脚本监听点击，每次改 `Image.sprite` 即可；最后播放 Animator 做淡出 → LoadScene。
* **ESC 退出**：把 EscapeQuit.cs 挂在 `GameManager`（或空对象 `System`）上，场景通用。
